# vehicle_nmpc

Эксперименты с RTI-NMPC-контроллером для гусеничного транспортного средства с
дифференциальным управлением. Проект собирает задачу оптимального управления в acados,
запускает замкнутую симуляцию на заданных траекториях, считает показатели качества и
может подбирать параметры контроллера через Optuna.

## Быстрый старт

Собрать Docker-образ с acados:

```bash
docker compose build
```

Запустить стандартную оценку контроллера:

```bash
docker compose up
```

По умолчанию используется режим `estimate`: контроллер прогоняется по настроенному
набору траекторий, а результаты сохраняются в `runs/...`.

Для запуска с параметрами из консоли удобно использовать `docker compose run`:

```bash
# Оптимизация параметров через Optuna
docker compose run --rm acados \
  python3 -m vehicle_nmpc.run runner.mode=optimize

# Только сгенерировать файлы решателя acados
docker compose run --rm acados \
  python3 -m vehicle_nmpc.run runner.mode=export

# Короткая проверка запуска
docker compose run --rm acados \
  python3 -m vehicle_nmpc.run runner.n_sim=20

# Переопределить параметр задачи управления
docker compose run --rm acados \
  python3 -m vehicle_nmpc.run problem.params.prediction_horizon_time=1.2
```

## Режимы запуска

- `estimate`: собирает модель, задачу, контроллер и симулятор, запускает все
  настроенные траектории, сохраняет графики и метрики.
- `optimize`: запускает попытки Optuna, затем отдельно прогоняет лучший найденный
  контроллер и сохраняет его результаты.
- `export`: собирает настроенные объекты acados и генерирует код решателя без
  замкнутой симуляции.

Результаты сохраняются в директорию запуска Hydra:

```text
runs/${runner.experiment_name}/${model.name}/${date}_${controller.name}/
```

Обычно внутри лежат:

- `acados/`: сгенерированный код acados и JSON-файлы.
- `plots/`: графики заданной и фактической траектории.
- `metrics/`: метрики по каждой траектории, сводка и данные оптимизации.
- `trials/`: результаты отдельных попыток Optuna.
- `best_estimate/`: финальная оценка с лучшими параметрами Optuna.

## Структура репозитория

```text
src/vehicle_nmpc/
  configs/          Hydra-конфиги.
  controller/       Интерфейсы контроллеров и реализация RTI-NMPC.
  experiments/      Запуск замкнутой симуляции и оптимизации.
  metrics/          Метрики слежения, быстродействия, управления и построение графиков.
  models/           Модели ТС и данные модели для acados.
  problem/          Построение задачи оптимального управления.
  sim/              Интерфейс симулятора и acados-симулятор.
  trajectory/       Источники заданных траекторий.
  utils/            Схема конфигов, фабрики, валидация, запуск, исключения.
runs/               Сгенерированные результаты экспериментов.
```

## Конфигурация

Точка входа использует Hydra:

```text
src/vehicle_nmpc/configs/config.yaml
```

Основной конфиг собирается из групп:

- `model`: модель ТС, сейчас `tracked_veh_kinematic`.
- `problem`: горизонт предсказания, веса функции стоимости, ограничения управления,
  настройки решателя и путь для сгенерированных файлов.
- `controller`: реализация контроллера, сейчас `rti_nmpc`.
- `sim`: реализация симулятора.
- `trajectories`: список траекторий для оценки.
- `optuna`: оптимизируемые параметры и компоненты целевой функции.
- `runner`: режим запуска, длина симуляции, имя эксперимента, выходная директория.

Любые значения Hydra можно переопределять из командной строки:

```bash
python3 -m vehicle_nmpc.run runner.n_sim=200
python3 -m vehicle_nmpc.run problem.params.state_weights='[30.0,30.0,3.0]'
python3 -m vehicle_nmpc.run trajectories.0.params.speed=0.7
```

При запуске через Docker переопределения нужно писать после `python3 -m vehicle_nmpc.run`.

### Добавление компонентов

Основные объекты создаются через небольшие реестры. Чтобы добавить новый
компонент:

1. Реализовать класс в соответствующем пакете.
2. Зарегистрировать его декоратором, например `@register_model(...)` или
   `@register_trajectory(...)`.
3. Добавить или обновить YAML-конфиг в `src/vehicle_nmpc/configs/...`.
4. Выбрать компонент через Hydra, изменив группу конфига или поле `name`.

Например, конфиг траекторий задаётся списком:

```yaml
- name: straight
  params:
    speed: 0.5
    heading: 0.0
```

Поле `name` выбирает зарегистрированный источник траектории, а `params` передаются в
его класс конфигурации.
